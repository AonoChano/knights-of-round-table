# Structured Analysis

## 适用场景

当任务需要先把问题拆清楚，再进入回答、规划或讨论时使用。

## 执行步骤

1. 明确任务目标与成功标准
2. 识别硬约束与软约束
3. 列出关键假设与未知项
4. 提出 2 到 4 条候选路径
5. 比较路径的代价、风险与收益
6. 输出推荐路径及理由

## 输出要求

- 先给结论，再给结构化依据
- 不把推测说成事实
- 必须显式指出仍不确定的部分
